# 修复素材滚动加载不触发的问题

## 问题根因

`CollaborationPanel.vue` 第 68 行的滚动容器依赖 `@scroll` 事件触发加载更多。当 `pageSize` 较小（如 2）时，2 条卡片总高度约 140px，远小于容器可用高度，不产生滚动条，`handleMaterialScroll` 永远不会执行。

## 修复方案

在 `CollaborationPanel.vue` 中，给滚动容器加一个 `ref`，当素材列表渲染完成后（通过 watch），检查容器是否可滚动。如果**不可滚动且还有更多数据**，自动 emit `load-more-materials`。

### [CollaborationPanel.vue](e:\job-project\collabedit-fe\src\lmComponents\collaboration\CollaborationPanel.vue)

1. 引入 `ref`, `watch`, `nextTick`
2. 给滚动容器加 `ref="scrollContainerRef"`
3. 添加 watch：当 `materials` 或 `materialLoading` 变化时，在 `nextTick` 后检查容器是否需要自动加载

```typescript
import { ref, watch, nextTick } from 'vue'

const scrollContainerRef = ref<HTMLElement | null>(null)

watch(
  () => [props.materials.length, props.materialLoading],
  () => {
    if (props.materialLoading) return
    if (props.materials.length >= props.materialTotal) return
    nextTick(() => {
      const el = scrollContainerRef.value
      if (el && el.scrollHeight <= el.clientHeight) {
        emit('load-more-materials')
      }
    })
  }
)
```

模板第 68 行加 ref：

```html
<div ref="scrollContainerRef" class="overflow-y-auto flex-1 custom-scrollbar -mx-2 px-2" @scroll="handleMaterialScroll">
```

只改这一个文件，不影响其他任何功能。
