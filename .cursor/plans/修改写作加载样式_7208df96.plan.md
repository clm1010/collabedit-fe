# 修改写作进入编辑器的加载样式

## 现状分析

两个页面的 `handleEdit` 函数中使用 `ElLoading.service` 创建全屏加载遮罩：

- [src/views/training/performance/index.vue](src/views/training/performance/index.vue) 第 540-544 行
- [src/views/template/management/index.vue](src/views/template/management/index.vue) 第 742-746 行

当前配置均为：

```typescript
ElLoading.service({
  lock: true,
  text: '正在校验权限...',
  background: 'rgba(0, 0, 0, 0.7)' // 黑色半透明背景
})
```

[index.html](index.html) 中 `.app-loading-item` 定义了一个双环旋转加载动画（外环顺时针 + 内环逆时针），视觉效果比 Element Plus 默认的圆圈 Loading 更好。

## 修改方案

### 1. 修改 `ElLoading.service` 配置

在两个文件的 `handleEdit` 中，修改 `ElLoading.service` 调用：

- `background` 改为 `'rgba(255, 255, 255, 0.7)'`
- 添加 `spinner` 属性，传入自定义 HTML 实现双环旋转动画（与 index.html 中 `.app-loading-item` 一致）
- 添加 `customClass` 属性用于样式控制（如需将 text 颜色改为深色以适配白色背景）

```typescript
ElLoading.service({
  lock: true,
  text: '正在校验权限...',
  background: 'rgba(255, 255, 255, 0.7)',
  spinner: `<div class="app-loading-item">
    <div class="app-loading-outter"></div>
    <div class="app-loading-inner"></div>
  </div>`,
  customClass: 'custom-writing-loading'
})
```

### 2. 创建共享 SCSS 文件 `src/lmStyles/loading.scss`

由于 `ElLoading` 的 DOM 是挂载到 `body` 上的，scoped 样式无法生效。参照项目中 `dialog.scss` 的模式，在 `src/lmStyles/` 下新建 `loading.scss`，包含：

- `.custom-writing-loading` 下的 `.app-loading-item` / `.app-loading-outter` / `.app-loading-inner` 布局与尺寸样式（复用 index.html 中的定义）
- `@keyframes loader-outter` 和 `@keyframes loader-inner` 动画
- `.custom-writing-loading .el-loading-text` 的文字颜色改为深色（适配白色背景）
- 隐藏默认的 `.el-loading-spinner .circular` SVG

### 3. 组件引用

在两个 Vue 组件的 `<style>` 中通过 `@use '@/lmStyles/loading.scss'` 引入样式。

## 需修改的文件

- **新建** `src/lmStyles/loading.scss` -- 自定义 loading 双环旋转动画样式
- `src/views/training/performance/index.vue` -- handleEdit 的 ElLoading 配置 + 引入 loading.scss
- `src/views/template/management/index.vue` -- handleEdit 的 ElLoading 配置 + 引入 loading.scss
