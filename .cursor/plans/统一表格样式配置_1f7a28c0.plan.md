# 统一 Element Plus 表格样式配置方案

## 背景

当前项目通过 `@use '@/lmStyles/table.scss'` 在每个使用 `el-table` 的组件中单独引入表格样式，维护分散。需要改为集中配置，一次引入全局生效。

当前 Element Plus 版本为 **2.11.1**，不支持 `setPropsDefaults`（需 2.13.3+）。采用手动修改 `ElTable` 组件 props 默认值的方式实现等效效果。

## 当前 table.scss 样式与 ElTable props 的映射

[table.scss](e:\job-project\collabedit-fe\src\lmStyles\table.scss) 中的样式可完全映射到 `ElTable` 的 props：

- 表头 thead 高度 64px -> `headerRowStyle: { height: '64px' }`
- 表头 th 背景色 #f0f4fa -> `headerCellStyle.backgroundColor`
- 表头 th 边框色 #8bb8ff -> `headerCellStyle.borderBottomColor`
- 表头 th 文字色 #333333 -> `headerCellStyle.color`

## 实现方案

### 1. 创建 `src/utils/elementplusTableConfig.ts`

通过直接修改 `ElTable` 组件对象的 props 默认值实现全局生效。由于 JS 模块是单例的，`unplugin-vue-components` 按需引入的 `ElTable` 和这里导入的是同一个对象，修改一次即全局生效：

```typescript
import { ElTable } from 'element-plus'

export function setupElementPlusTableConfig() {
  const tableProps = ElTable.props as Record<string, any>

  if (tableProps.headerRowStyle) {
    tableProps.headerRowStyle.default = () => ({ height: '64px' })
  }

  if (tableProps.headerCellStyle) {
    tableProps.headerCellStyle.default = () => ({
      backgroundColor: '#f0f4fa',
      borderBottomColor: '#8bb8ff',
      color: '#333333'
    })
  }
}
```

原理：

- `ElTable` 组件的 `header-row-style` 和 `header-cell-style` 是官方文档记录的 props（参考 [Element Plus Table](https://element-plus.org/zh-CN/component/table.html)）
- 直接修改 props 的 `default` 值，等同于 2.13.3 的 `setPropsDefaults` 效果
- 不使用任何 CSS，纯 TypeScript 实现
- 各组件仍可通过单独传入 `header-row-style` / `header-cell-style` 来覆盖默认值

### 2. 在 main.ts 中引入

在 [main.ts](e:\job-project\collabedit-fe\src\main.ts) 的 `setupAll()` 函数中，**在 `setupElementPlus(app)` 之前**调用（确保在组件渲染前修改默认值）：

```typescript
import { setupElementPlusTableConfig } from '@/utils/elementplusTableConfig'

const setupAll = async () => {
  const app = createApp(App)

  setupElementPlusTableConfig() // 在 Element Plus 初始化之前设置

  await setupI18n(app)
  setupStore(app)
  // ...
}
```

### 3. 移除各组件中的 table.scss 引入

需要修改以下 3 个文件，删除 `@use '@/lmStyles/table.scss';`：

- [template/management/index.vue](e:\job-project\collabedit-fe\src\views\template\management\index.vue) -- 第 1317 行
- [training/performance/index.vue](e:\job-project\collabedit-fe\src\views\training\performance\index.vue) -- 第 1103 行
- [training/performance/components/ExamRecordDialog.vue](e:\job-project\collabedit-fe\src\views\training\performance\components\ExamRecordDialog.vue) -- 第 70 行

## 注意事项

- `DrillSelector.vue` 之前未引入 `table.scss`，全局配置后它也会自动获得统一的表头样式
- 各组件如需特殊表头样式，仍可通过 `:header-cell-style` prop 单独覆盖
- `table.scss` 文件保留不删除（避免其他可能的隐性引用），但不再被使用
- 后续升级到 Element Plus 2.13.3+ 后，可改用官方 `setPropsDefaults` API
