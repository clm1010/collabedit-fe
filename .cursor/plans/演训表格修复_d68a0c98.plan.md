# 演训表格字典与时间格式化修复

## 问题一：DrillSelector.vue 表格字典未转换

文件：`[src/views/training/performance/components/DrillSelector.vue](src/views/training/performance/components/DrillSelector.vue)`

当前第99-103行，演训类型/等级/学院列直接用 `prop` 展示，显示原始码值：

```html
<el-table-column label="演训类型" prop="exerciseType" width="120" align="center" />
<el-table-column label="演训等级" prop="level" width="100" align="center" />
<el-table-column label="演训学院" prop="academy" width="100" align="center" />
```

**修改方案：**

1. 补充缺失的 `levelOptions`（与 `index.vue` 保持一致：`ZLJ/YXJ/ZSJ`，注意当前文件内 `el-option` 中战役级用的是 `ZYJ`，需统一为 `YXJ`）
2. 为三列添加 `#default` 插槽，调用 `find` 查找 label 显示

## 问题二：多处时间字段未格式化（使用 dayjs）

全部采用统一格式 `YYYY-MM-DD HH:mm:ss`，带空值保护：

```html
<template #default="scope">
  {{ scope.row.xxxTime ? dayjs(scope.row.xxxTime).format('YYYY-MM-DD HH:mm:ss') : '' }}
</template>
```

## 影响评估（不影响现有功能）

- 所有改动仅在 `el-table-column` 上新增 `#default` 插槽用于**展示格式化**，不修改任何数据、API 调用或业务逻辑
- 加入空值保护 `? ... : ''`，后端返回 `null`/空字符串时不会报错
- `dayjs` 已在 `package.json` 中声明，无需新增依赖
- 每个文件仅需在 `<script setup>` 顶部新增一行 `import dayjs from 'dayjs'`
