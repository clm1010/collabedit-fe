# 复制内容格式修复

## 根因分析

`copyContent`（[TiptapCollaborativeEditor.vue](e:\job-project\collabedit-fe\src\views\training\document\TiptapCollaborativeEditor.vue) 第 519-529 行）将 HTML 通过 `innerText` 转为**纯文本**后写入剪贴板：

```typescript
const copyContent = async (html: string) => {
  const tempDiv = document.createElement('div')
  tempDiv.innerHTML = html
  const text = tempDiv.innerText || tempDiv.textContent || ''
  const ok = await copyToClipboard(text)  // 只写 text/plain
}
```

底层 `copyToClipboard`（[clipboard.ts](e:\job-project\collabedit-fe\src\views\utils\clipboard.ts)）也只支持纯文本（`writeText` / `textarea + execCommand`）。

手动选中 v-html 渲染内容后 Ctrl+C 时，浏览器自动写入 `text/html` + `text/plain` 两种格式，Tiptap 能正确解析 HTML。

## 修复方案

涉及 2 个文件，改动量很小。

### 1. `clipboard.ts` — 新增 `copyHtmlToClipboard` 函数

在 [src/views/utils/clipboard.ts](e:\job-project\collabedit-fe\src\views\utils\clipboard.ts) 中新增一个支持 HTML 格式的复制函数：

- 优先使用 `navigator.clipboard.write` + `ClipboardItem`，同时写入 `text/html` 和 `text/plain`
- 降级方案：创建一个隐藏 div 设置 `innerHTML`，通过 `Selection` + `execCommand('copy')` 复制（浏览器会自动带上 HTML 格式）

### 2. `TiptapCollaborativeEditor.vue` — 改造 `copyContent`

将第 519-529 行的 `copyContent` 改为调用新的 `copyHtmlToClipboard`，直接传入原始 HTML 字符串，不再预先转为纯文本。
