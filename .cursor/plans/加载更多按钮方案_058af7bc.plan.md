# 增加"加载更多"按钮（与滚动加载并存）

只改 [CollaborationPanel.vue](e:\job-project\collabedit-fe\src\lmComponents\collaboration\CollaborationPanel.vue)。

## 问题

`pageSize` 较小时，卡片总高度不足以撑满 `el-splitter-panel` 的 60% 区域，`overflow-y-auto` 容器无滚动条，`@scroll` 不触发，无法加载更多。

## 方案

在第 85-86 行区域（"加载中"和"已加载全部"提示之间），增加一个"加载更多"按钮。当还有未加载的数据且不在加载中时显示，点击 emit `load-more-materials`。

### 模板改动（第 85-86 行）

```html
<div v-if="materialLoading" class="text-center text-gray-400 py-2 text-xs">加载中...</div>
<template v-else-if="materials.length < materialTotal">
  <div class="text-center py-2">
    <el-button text size="small" type="primary" @click="emit('load-more-materials')">加载更多</el-button>
  </div>
</template>
<div v-else class="text-center text-gray-400 py-2 text-xs">已加载全部</div>
```

逻辑说明：

- `materialLoading` 为 true -> 显示"加载中..."
- 还有更多数据（`materials.length < materialTotal`）-> 显示"加载更多"按钮
- 数据已全部加载 -> 显示"已加载全部"

滚动加载（`@scroll="handleMaterialScroll"`）保持不变，两种方式并存：

- 内容不够高无滚动条 -> 用户点按钮
- 内容够高有滚动条 -> 滚动触底自动加载，同时按钮也可点
