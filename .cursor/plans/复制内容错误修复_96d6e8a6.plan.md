# 参考素材"复制内容"按钮 TypeError 修复

## 错误分析

### 根本原因

截图中的错误为：`TypeError: Cannot read properties of undefined (reading 'writeText')`

错误发生在 [TiptapCollaborativeEditor.vue](src/views/training/document/TiptapCollaborativeEditor.vue) 第 469 行的 `copyContent` 函数中：

```463:477:e:\job-project\collabedit-fe\src\views\training\document\TiptapCollaborativeEditor.vue
const copyContent = (html: string) => {
  const tempDiv = document.createElement('div')
  tempDiv.innerHTML = html
  const text = tempDiv.innerText || tempDiv.textContent || ''

  navigator.clipboard           // <-- navigator.clipboard 为 undefined!
    .writeText(text)
    .then(() => {
      ElMessage.success('内容已复制到剪贴板')
    })
    .catch(() => {
      ElMessage.error('复制失败')
    })
}
```

**问题链条：**

1. 应用通过 `http://192.168.1.70:5173` 访问（非 HTTPS、非 localhost）
2. `navigator.clipboard` API 仅在**安全上下文**（Secure Context：HTTPS 或 localhost）下可用
3. 在非安全上下文中，`navigator.clipboard` 为 `undefined`
4. 代码直接调用 `navigator.clipboard.writeText()`，触发同步 TypeError
5. 代码中的 `.catch()` 仅能捕获 Promise 的异步拒绝，**无法捕获**同步抛出的 TypeError

### 同项目已有的正确参考实现

- [LinkPopover.vue](src/views/training/document/components/toolbar/LinkPopover.vue)（第 166-188 行）：先检测 `navigator.clipboard && navigator.clipboard.writeText`，不可用时降级 `document.execCommand('copy')`
- [infra/file/index.vue](src/views/infra/file/index.vue)（第 190 行）：检测 `navigator.clipboard && window.isSecureContext`

## 修复方案

### 1. 提取公共的剪贴板兼容工具函数

在 `src/views/utils/clipboard.ts`（或类似公共 utils 位置）创建 `copyToClipboard(text: string): Promise<boolean>` 工具函数：

- 优先检查 `navigator.clipboard && navigator.clipboard.writeText`，可用则使用
- 不可用则降级到 `document.execCommand('copy')` + 隐藏 textarea
- 整体包裹在 try-catch 中，保证不会抛出未捕获异常
- 返回 `Promise<boolean>` 表示成功/失败，由调用方决定提示文案

### 2. [严重] 修复 `TiptapCollaborativeEditor.vue` 的 `copyContent` 函数

替换第 463-477 行，使用公共工具函数。这是唯一导致页面崩溃的问题，优先级最高。

### 3. [中等] 修复编辑器相关文件

改造 `ExportToolbar.vue`、`TiptapEditor.vue`、`MarkdownEditor.vue` 中直接调用 `navigator.clipboard.writeText` 的地方，统一使用公共工具函数。

### 4. [低] 修复 IoT 模块文件

改造 `ProductDetailsHeader.vue`、`DeviceDetailsHeader.vue`、`DeviceDetailsInfo.vue` 中的 `copyToClipboard` 方法，统一使用公共工具函数。

## 影响评估

- **安全上下文（HTTPS / localhost）**：行为完全不变，仍走 Clipboard API
- **非安全上下文（HTTP 局域网）**：从"崩溃/失败"变为通过 execCommand 降级正常复制
- **不涉及**：响应式数据、组件结构、编辑器核心、事件通信、样式
- **风险等级**：极低 -- 仅增加兼容性降级路径，不修改现有成功路径
